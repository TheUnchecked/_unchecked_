BloodHound CE su Kali Linux:
guida rapida al primo avvio.

Hai installato BloodHound CE con Docker. Adesso vuoi capire come avviarlo, raccogliere i dati dal dominio e leggere i risultati senza perdere un'ora a inseguire errori stupidi. Questa guida va dritta al punto.

BloodHound Active Directory Kali Linux Red Team

Avvio di BloodHound

La prima volta, scarica il file di configurazione e avvia i container:

1mkdir ~/bloodhound && cd ~/bloodhound
2curl -L https://ghst.ly/getbhce -o docker-compose.yml
3sudo docker compose up

Nei log trovi la password iniziale dell'utente admin:

1sudo docker compose logs bloodhound | grep -i password

Accedi su http://localhost:8080/ui/login con utente admin e la password trovata nei log. Al primo accesso ti verrà chiesto di cambiarla.

Dagli avvii successivi ti basta:

1cd ~/bloodhound
2sudo docker compose up

Per spegnere:

1sudo docker compose down

Configurazione di rete

Se la tua VM usa un'interfaccia eth1 per raggiungere il dominio, tieni presente che la configurazione IP non è persistente: si perde a ogni riavvio della VM e va riassegnata manualmente.

1sudo ip addr add 192.168.10.50/24 dev eth1
2sudo ip link set eth1 up
3echo "nameserver 192.168.10.10" | sudo tee /etc/resolv.conf

Verifica sempre la connettività prima di lanciare la scansione. Se non risponde, l'IP è saltato — riassegna e riprova.

1ping -c 3 192.168.10.10

Raccolta dati con bloodhound-python

Installa il collector:

1pip install bloodhound --break-system-packages

Crea una cartella dedicata e lancia la scansione:

1mkdir ~/bloodhound-data && cd ~/bloodhound-data
2
3bloodhound-python -u <utente> -p <password> -d staging.lab -ns 192.168.10.10 -c all
-u / -p
Credenziali utente di dominio
Funziona anche con un account senza privilegi. Non serve essere Domain Admin per raccogliere i dati.
-d
Nome del dominio
Il nome FQDN del dominio Active Directory da analizzare. Es: staging.lab
-ns
IP del Domain Controller
L'indirizzo IP del DC che funge da nameserver per il dominio.
-c all
Raccoglie tutti i dati disponibili
Utenti, gruppi, computer, sessioni, ACL, trust. I file .json vengono salvati nella cartella corrente.

Tieni la cartella ordinata: se lanci più scansioni, i file si accumulano e puoi perdere traccia di quale set è aggiornato.

Importazione dati e utilizzo dell'interfaccia

Apri http://localhost:8080/ui/login, clicca sull'icona di upload nella barra sinistra e seleziona tutti i file .json da ~/bloodhound-data. L'importazione richiede qualche secondo.

Vai su Explore e cerca il dominio nella barra di ricerca. Da lì puoi navigare la struttura del dominio e lanciare le query predefinite dall'icona con il fulmine.

Query di attacco

Shortest Paths to Domain Admins

List all Kerberoastable Users

List all AS-REP Roastable Users

Query di ricognizione

Find Computers with Sessions

Find all Domain Admins

Find Shortest Paths to High Value Targets

Per query più avanzate: queries.specterops.io

Errori comuni

!
Porta 7474 già in uso
Messaggio: "failed to bind host port 127.0.0.1:7474". Causa: Neo4j installato sul sistema occupa la porta. Soluzione: sudo systemctl stop neo4j
!
DNS timeout durante la scansione
Messaggio: "dns.resolver.LifetimeTimeout". Causa: eth1 ha perso l'IP o il DC non è raggiungibile. Verifica connettività e riassegna l'IP se necessario.
!
dhclient non trovato
Usa dhcpcd al posto di dhclient: sudo dhcpcd eth0

Reinstallazione completa

Se qualcosa è andato storto e vuoi ripartire da zero:

1# Rimuovi container e volumi
2sudo docker compose down -v
3sudo docker system prune -a --volumes
4
5# Rimuovi Neo4j dal sistema
6sudo apt remove --purge neo4j python3-neo4j -y
7sudo apt autoremove -y
8sudo rm -rf /var/lib/neo4j /var/log/neo4j /etc/neo4j
9
10# Reinstalla
11curl -L https://ghst.ly/getbhce -o docker-compose.yml
12sudo docker compose up
