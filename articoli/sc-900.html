Microsoft SC-900 Security Identity Compliance Certificazione

Il 9 maggio sostengo l'esame SC-900 — Microsoft Security, Compliance and Identity Fundamentals. Ho costruito questo sistema di studio per non limitarmi a prendere appunti ma per ritenere davvero quello che studio. È pubblico perché potrebbe essere utile anche a te.

L'esame richiede un punteggio minimo di 700 su 1000. Non chiede di saper configurare nulla: chiede di capire cosa fa ogni strumento Microsoft nell'ecosistema di sicurezza cloud, e perché esiste. La distinzione è importante, perché cambia completamente il modo in cui si studia.

Non memorizzare. Costruisci una mappa mentale di come i prodotti Microsoft si relazionano tra loro. L'esame ragiona sempre per scenari.

I 5 principi che reggono tutto

Questi non sono concetti da memorizzare: sono le lenti attraverso cui leggere ogni domanda dell'esame.

🏰
Difesa in profondità
La sicurezza non è un muro unico ma una serie di strati sovrapposti. Ogni strato assume che il precedente possa fallire: Fisica → Identità → Perimetro → Rete → Calcolo → Applicazione → Dati.
🔐
Zero Trust — non fidarsi mai, verificare sempre
Nessun utente, dispositivo o rete è affidabile per definizione. Ogni accesso richiede verifica esplicita. Principio del minimo privilegio. Assumi sempre la violazione.
🪪
L'identità è il nuovo perimetro
In un mondo cloud non esiste un confine fisico da difendere. L'identità — utente, dispositivo, applicazione — è l'unico controllo perimetrale rimasto.
⚖️
Responsabilità condivisa
Microsoft gestisce l'infrastruttura fisica; il cliente rimane sempre responsabile dei propri dati, degli accessi e delle identità. Il confine si sposta in base al modello: IaaS, PaaS o SaaS.
🎛️
Autenticazione ≠ Autorizzazione
Autenticazione = sei chi dici di essere? (chi sei). Autorizzazione = hai il permesso di fare questo? (cosa puoi fare). Due fasi distinte e sequenziali, sempre.
Sicurezza = Prevenzione + Rilevamento + Risposta
Zero Trust = Verifica esplicita + Minimo privilegio + Assumi la violazione
Identità = Autenticazione (chi) + Autorizzazione (cosa) + Governance (quanto)

I 10 concetti che valgono l'80%

Se dovessi studiare solo questi dieci argomenti, probabilmente passeresti l'esame lo stesso. Padroneggiali prima di tutto il resto.

01
Zero Trust — i tre principi
Verifica esplicita · Minimo privilegio · Assumi la violazione. La domanda preferita dell'esame: «quale principio Zero Trust si applica a questo scenario?»

02
Microsoft Entra ID — tipi di identità
Utenti · Utenti guest · Gruppi · Entità servizio · Identità gestite. Sapere cosa è ciascuna e perché esiste.
25–30%
03
MFA — autenticazione a più fattori
Qualcosa che sai · Qualcosa che hai · Qualcosa che sei. Il controllo di sicurezza con il miglior rapporto costo/beneficio in assoluto.
25–30%
04
Accesso condizionale
Il motore IF-THEN di Microsoft: SE (utente + posizione + dispositivo + app), ALLORA (consenti / blocca / richiedi MFA). È l'implementazione pratica di Zero Trust.
25–30%
05
RBAC — controllo degli accessi in base al ruolo
Si assegnano ruoli, non permessi individuali. Azure RBAC (risorse) vs Entra RBAC (directory) — distinzione spesso testata.
25–30%
06
Microsoft Defender XDR — l'ecosistema
Defender for Office 365 (email) · Defender for Endpoint (dispositivi) · Defender for Identity (AD on-prem) · Defender for Cloud Apps (CASB). Sapere quale protegge cosa.
35–40%
07
Microsoft Sentinel — SIEM + SOAR
SIEM = raccoglie e analizza log. SOAR = automatizza le risposte agli incidenti. Sentinel fa entrambe le cose su scala cloud-native.
35–40%
08
Defender for Cloud — CSPM
Gestione del profilo di sicurezza cloud (Cloud Security Posture Management). Rileva configurazioni errate. Fornisce un Secure Score. Copre ambienti multi-cloud.
35–40%
09
Microsoft Purview — DLP + etichette di riservatezza
DLP impedisce la fuga di dati sensibili. Le etichette classificano e proteggono i documenti. Purview unifica governance dei dati e conformità normativa.
20–25%
10
Compliance Manager + punteggio di conformità
Strumento di valutazione della conformità normativa. Il punteggio misura il progresso verso gli standard (GDPR, ISO 27001). Non è una certificazione automatica.
20–25%
